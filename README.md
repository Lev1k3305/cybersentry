# 🛡️ CyberSentry & Центральный Командный Центр (ЦКЦ)

Добро пожаловать в репозиторий проекта **CyberSentry** и **ЦКЦ** — современного монорепозитория, объединяющего B2C-платформу кибербезопасности, интерактивный командный терминал и мобильное приложение.

Проект разработан с учетом строгих стандартов безопасности (OSINT/Anti-Fraud), высокой производительности и интерактивного UI/UX в стиле киберпанк.

---

## 📂 Структура проекта (Monorepo)

Репозиторий организован как монорепозиторий на базе **pnpm workspaces**. Он разделен на независимые приложения (артефакты) и общие библиотеки:

```text
├── artifacts/                         # Основные приложения и сервисы
│   ├── cybersentry/                   # Веб-панель CyberSentry (React + Vite + Tailwind)
│   ├── cybersentry-api/               # Python FastAPI микросервис (порт 8000) для OSINT-проверок
│   ├── command-console/               # Центральный Командный Центр (ЦКЦ) — терминальный UI на React
│   ├── api-server/                    # Node.js Express API Gateway (порт 8080)
│   ├── mobile/                        # Мобильное приложение CyberSentry (Expo / React Native)
│   └── mockup-sandbox/                # Песочница для макетирования интерфейсов
│
├── lib/                               # Общие библиотеки и конфигурации
│   ├── api-spec/                      # OpenAPI-спецификация (openapi.yaml) и конфиг Orval
│   ├── api-client-react/              # Сгенерированные React Query хуки для API
│   ├── api-zod/                       # Сгенерированные схемы валидации Zod
│   └── db/                            # Слой работы с базой данных и Drizzle ORM
│
└── .jules/                            # База знаний и журналы разработки (Sentinel, Bolt, Palette)
```

## 🛠️ Технологический стек

- **Frontend & Web:** React 19, Vite, Tailwind CSS (с кастомными киберпанк-палитрами), Framer Motion, TanStack React Query, Wouter.
- **Mobile:** Expo 54, React Native 0.81, Expo Router, Reanimated.
- **Backend (Node.js):** Express 5 (API-шлюз и диспетчер системных команд ЦКЦ).
- **Backend (Python):** Python 3.11, FastAPI, Uvicorn, HTTPX (асинхронный OSINT-бэкенд для CyberSentry).
- **Базы данных & Валидация:** Drizzle ORM, Zod, Pydantic.
- **Инструменты генерации API:** Orval (автогенерация клиентов и Zod-схем из OpenAPI Spec).

## 🚀 Быстрый старт и запуск

Перед началом работы убедитесь, что у вас установлены **Node.js (v24+)**, **pnpm** и **Python 3.11**.

### 1. Установка зависимостей
Установите все зависимости монорепозитория одной командой:
```bash
pnpm install
```

### 2. Запуск локального окружения
Для одновременного запуска ключевых компонентов системы:

- **API Gateway (Express):**
  ```bash
  pnpm --filter @workspace/api-server run dev
  ```
  *Сервер запускается на порту `8080` и проксирует запросы на Python бэкенд.*

- **Интерфейс Командного Центра (ЦКЦ):**
  ```bash
  pnpm --filter @workspace/command-console run dev
  ```

- **Веб-панель CyberSentry:**
  ```bash
  pnpm --filter @workspace/cybersentry run dev
  ```

- **Асинхронный Python API (FastAPI):**
  ```bash
  cd artifacts/cybersentry-api
  pip install -r requirements.txt
  python -m uvicorn main:app --host 0.0.0.0 --port 8000
  ```

- **Мобильное приложение (Expo):**
  ```bash
  pnpm --filter @workspace/mobile run dev
  ```

### 3. Сборка и проверка типов
- **Полная проверка типов по всему репозиторию:**
  ```bash
  pnpm run typecheck
  ```
- **Сборка всех пакетов:**
  ```bash
  pnpm run build
  ```
- **Регенерация API-клиента из OpenAPI-спецификации:**
  При внесении изменений в `lib/api-spec/openapi.yaml` запустите:
  ```bash
  pnpm --filter @workspace/api-spec run codegen
  ```

## 🛡️ Безопасность и архитектурные решения

Проект CyberSentry и ЦКЦ спроектирован с учетом производительности и лучших практик веб-безопасности:

### 1. Строгий контракт интеграции внешних API (Fallback Contract)
Для проверки репутации телефонов и утечек email используются внешние провайдеры (numlookupapi.com и HIBP v3). Архитектура строго разграничивает демонстрационный и рабочий режимы:
- **Режим Demo:** Активируется только при отсутствии ключей в окружении (`PHONE_API_KEY` или `BREACH_API_KEY`). Результаты помечаются флагом `source: "demo"` с добавлением локальных эвристик и базы знаний.
- **Режим Production:** Если ключи заданы, бэкенд обращается к внешним сервисам. **В случае сбоя внешнего API система возвращает HTTP 502 Bad Gateway вместо тихого перехода на демонстрационные данные**, гарантируя прозрачность и корректность принимаемых защитных решений.

### 2. Кросс-стековая валидация входных данных (SSoT)
- Единым источником правды (**Single Source of Truth**) является спецификация `lib/api-spec/openapi.yaml`.
- На основе спецификации автоматически генерируются Zod-схемы для валидации на фронтенде.
- В Python-микросервисе Pydantic-модели (`PhoneInput`, `EmailInput`) дублируют данные ограничения (регулярные выражения, длина строк), защищая систему от SSRF, Path Traversal и SQL/NoSQL инъекций.

### 3. Оптимизация производительности (Bolt Engine)
- **Пул соединений HTTPX (Connection Pooling):** Python-микросервис инициализирует единый глобальный `httpx.AsyncClient` внутри контекстного менеджера `lifespan` FastAPI. Это исключает накладные расходы на повторные TCP-подключения и TLS-рукопожатия при внешних запросах.
- **Оптимизация рендеринга логов и терминала (React.memo):**
  - Списки логов безопасности (вкладка «Журнал событий»), обновляющиеся по механизму polling (раз в 5-10 секунд), используют `React.memo` с кастомным компаратором идентификаторов (`prev.entry.id === next.entry.id`). Это предотвращает ресурсоемкий парсинг дат и локализацию времени на каждом тике опроса.
  - Компоненты строк терминала мобильного приложения (`EntryRow`) также мемоизированы. Это устраняет избыточный рендеринг всего списка сообщений при анимации мерцающего курсора (частота 500 мс).

### 4. UI/UX Палитра и Доступность
Интерфейс выполнен в единой темно-сланцевой теме (`#111827`) с акцентными цветами индикации состояний:
- 🟢 **Изумрудный зеленый (`#10B981`)** — Безопасное состояние (Safe).
- 🟡 **Янтарный желтый (`#F59E0B`)** — Предупреждение (Warning).
- 🔴 **Неоново-розовый / красный (`#EF4444`)** — Опасность / Нарушение (Danger/Alert).

В соответствии с правилами доступности, для конфиденциальных поисков в CyberSentry внедрена кнопка быстрой очистки запроса. Она одновременно сбрасывает поле ввода, инвалидирует кэш TanStack Query и переводит фокус на поле ввода, предотвращая утечку данных на экране («shoulder-surfing»).

## 🔑 Переменные окружения (Secrets)

Для работы в режиме интеграции с внешними провайдерами укажите следующие переменные окружения (например, через Replit Secrets или файл `.env`):

- `PHONE_API_KEY` — Ключ авторизации для [numlookupapi.com](https://numlookupapi.com) (проверка номеров).
- `BREACH_API_KEY` — API-ключ для [Have I Been Pwned](https://haveibeenpwned.com) (сканер утечек аккаунтов).
- `PORT` — Порт для запуска веб-сервера (по умолчанию 3000 / 8080 для API).
- `EXPO_PUBLIC_DOMAIN` — Домен хоста для настройки глубоких ссылок и сетевых подключений мобильного клиента.
